SwagBounties
Full-featured player bounty system with GUI, Vault economy integration, and Discord webhooks.
Spigot 1.21.4 Java 21 Vault Economy Discord Webhooks
Core Features
💰
Place & Stack Bounties
Place bounties on online or offline players with `/bounty set`, or add to an existing one with `/bounty add`. Anonymous flag, min/max limits, and placement tax included.
⚔️
Claim on Kill
Rewards are automatically deposited via Vault when a target is killed. Same-IP exploit prevention blocks self-farming across accounts.
🖼️
Paginated GUI
Skull-based chest GUI showing every wanted player with total reward. Click any target for a full detail view — individual bounties, amounts, and placer info.
⏰
Expiry & Notifications
Unclaimed bounties auto-expire with a partial refund. Players are warned on login if they have bounties on their head. `/bounty top` shows the 5 most wanted.
Economy System
Vault Integration
Configurable Tax System
Two independent tax rates — placement tax deducted when a bounty is placed, and expiry refund tax deducted when a bounty is refunded on expiry. Set min/max bounty limits server-wide. All transactions go through Vault for full economy plugin compatibility.

Integration & Tooling
📣
Discord Webhooks
Post bounty set, claim, and expiry events to a Discord channel. Configurable threshold — only notify on high-value bounties.
📊
PlaceholderAPI
5 placeholders for scoreboards, chat formatters, and any PAPI-compatible plugin. Top target, top reward, total bounties, and per-player lookups.
🛡️
Admin Commands
9 admin subcommands — force-remove, wipe, inspect anonymous bounties, place server bounties, and edit config values live without restarting.
